# Claude Code로 PPT 100% 자동화하는 방법 (2025 완벽 가이드)

PPT 만드는 데 3시간씩 써본 적 있으신가요? 자료 조사하고, 내용 정리하고, 디자인하고... 정작 중요한 일은 못하고 반복 작업에만 시간을 쏟는 경험, 누구나 있을 거예요.

하지만 2025년, 이제 그럴 필요가 없어요. Claude Code를 활용하면 자료조사부터 PPT 파일 생성까지 전 과정을 AI 에이전트 팀이 자동으로 처리해 줍니다. 사람은 최종 검토만 하면 되죠.

이 글에서는 Claude Code로 PPT를 100% 자동화하는 방법을 단계별로 알려드릴게요. SubAgent와 Skills 개념부터 실제 팀 구성, 실습 과정까지 모두 담았어요.

---

## Claude Code는 코딩 도구가 아니라 '자동화 오케스트레이터'예요

많은 사람들이 Claude Code를 단순한 바이브 코딩 도구로만 알고 있어요. 하지만 실제로는 n8n, Make 같은 자동화 플랫폼보다 더 유연하고 강력한 도구가 될 수 있답니다.

**Claude Code의 진짜 장점은 이거예요:**

- 자연어로 워크플로우 구성 가능
- 에이전트끼리 자율적으로 소통하며 작업
- 복잡한 판단이 필요한 업무도 처리 가능
- CLI 기반이라 다양한 도구와 연동 쉬움

n8n이나 Make는 미리 정의된 노드를 연결하는 방식이라 새로운 작업이 생기면 매번 워크플로우를 수정해야 해요. 하지만 Claude Code는 자연어로 지시만 하면 AI가 알아서 방법을 찾아내죠. 복합적인 작업일수록 그 차이가 확실히 느껴져요.

> 💡 **Tip**: Claude Code는 단순 반복 작업보다 '여러 에이전트가 협업해야 하는 복잡한 프로젝트'에서 진가를 발휘해요.

## 정규직(SubAgent) vs 프리랜서(Skills) 개념 이해하기

Claude Code로 자동화 팀을 구성하려면 먼저 두 가지 개념을 알아야 해요. SubAgent와 Skills인데, 빌더조쉬는 이걸 아주 쉽게 비유했어요.

### SubAgent = 정규직 직원

복잡한 판단이 필요한 업무를 맡기는 에이전트예요. 사람으로 치면 정규직 직원 같은 존재죠.

**특징:**
- 상황에 따라 스스로 판단하고 행동
- 여러 단계의 추론과 의사결정 수행
- 토큰 소모량이 높음 (생각을 많이 하니까요)
- 예시: 리서치 에이전트, 콘텐츠 정리 에이전트

**언제 사용하나요?**
- 검색 결과를 분석해 핵심 정보 추출할 때
- 여러 자료를 종합해 요약문 작성할 때
- 상황별로 다른 전략이 필요한 업무

### Skills = 프리랜서

매뉴얼대로 반복 수행하는 단순 업무를 맡기는 도구예요. 프리랜서처럼 특정 작업만 처리하죠.

**특징:**
- 정해진 규칙대로만 작동
- 판단 없이 빠르게 실행
- 토큰 소모량이 낮음 (생각 안 하고 바로 실행)
- 예시: 디자인 스킬, PPTX 파일 생성 스킬

**언제 사용하나요?**
- 디자인 템플릿 적용할 때
- PPT 파일 생성할 때
- 정해진 포맷으로 데이터 변환할 때

| 구분 | SubAgent | Skills |
|------|----------|--------|
| 비유 | 정규직 직원 | 프리랜서 |
| 역할 | 복잡한 판단 업무 | 단순 반복 업무 |
| 토큰 소모 | 높음 | 낮음 |
| 활용 예시 | 리서치, 분석, 정리 | 디자인, 파일 생성 |

이 구분을 이해하면 토큰 비용도 아끼고 작업 속도도 빨라져요. 판단이 필요한 일만 SubAgent에게 맡기고, 나머지는 Skills로 처리하면 되니까요.

## PPT 자동화 팀 구성하는 방법

이제 실전으로 들어가볼까요? PPT를 100% 자동으로 만들려면 3개 파트로 팀을 구성해야 해요.

### 1. 기획 팀 (SubAgent)

자료를 찾고 정리하는 역할이에요. 판단이 필요하니 SubAgent로 구성하죠.

**리서치 에이전트**
- 웹 검색으로 관련 자료 수집
- 신뢰도 높은 정보 선별
- 최신 데이터 우선 수집

**정리 에이전트**
- 수집한 정보를 PPT 구조로 정리
- 핵심 메시지 추출
- 슬라이드별 내용 구성

### 2. 디자인 팀 (Skills)

브랜드 가이드라인에 맞춰 디자인하는 역할이에요. 정해진 규칙대로 하니 Skills로 충분해요.

**frontend-design 스킬**
- 폰트, 컬러, 레이아웃 적용
- 브랜드 아이덴티티 유지
- 시각적 일관성 확보

### 3. 개발 팀 (Skills)

실제 PPT 파일을 생성하는 역할이에요. Anthropic이 공식 제공하는 스킬을 사용하면 돼요.

**pptx 스킬**
- Anthropic 공식 GitHub에서 제공
- JSON 데이터를 PPTX 파일로 변환
- 이미지, 차트, 표 자동 삽입

> 💡 **Tip**: Anthropic 공식 GitHub에서 frontend-design과 pptx 스킬을 모두 제공해요. 직접 만들 필요 없이 가져다 쓰면 됩니다.

**팀 협업의 진짜 가치**

이 시스템의 장점은 팀 전체가 같은 품질의 결과물을 만들 수 있다는 거예요. 브랜드 가이드라인, 폰트, 템플릿을 한 번만 설정해 두면 누가 만들든 똑같은 퀄리티가 나와요. 개인보다 팀에서 훨씬 강력한 도구가 되는 이유죠.

## 실제로 PPT 자동화 시스템 구축하기

구체적인 세팅 방법을 알려드릴게요. Cursor와 Claude Code만 있으면 바로 시작할 수 있어요.

### 1단계: 환경 세팅

**필요한 도구**
- Cursor (코드 에디터)
- Claude Code (CLI 터미널 도구)
- Git (스킬 및 에이전트 다운로드용)

터미널에서 Claude Code를 실행하면 자연어로 지시를 내릴 수 있어요. 코딩 없이도 자동화 워크플로우를 만들 수 있답니다.

### 2단계: SubAgent와 Skills 문서 작성

각 에이전트와 스킬이 무슨 일을 해야 하는지 문서로 정리해야 해요. 프롬프트 형태로 작성하면 되는데, 핵심은 이거예요:

**SubAgent 프롬프트 작성법**
- 에이전트의 역할과 목표 명확히 정의
- 판단 기준 제시 (예: 신뢰도 높은 출처 우선)
- 다른 에이전트와 협업 방식 명시

**Skills 프롬프트 작성법**
- 입력 데이터 형식 정의
- 처리 규칙 단계별로 나열
- 출력 형식 명확히 지정

> 💡 **Tip**: Claude Code의 '플랜 모드'를 활용하면 프롬프트 초안을 AI가 작성해 줘요. 수정만 하면 되니 훨씬 빨라요.

### 3단계: Anthropic 공식 Skills 연동

Anthropic GitHub에서 제공하는 스킬을 가져와요.

```
git clone https://github.com/anthropics/claude-code-skills
```

**연동할 스킬**
- frontend-design: 디자인 가이드라인 적용
- pptx: PowerPoint 파일 생성

이 스킬들은 이미 잘 만들어져 있어서 바로 사용 가능해요. 커스터마이징도 쉽게 할 수 있고요.

### 4단계: 실제 PPT 생성 테스트

자, 이제 실제로 PPT를 만들어볼까요? 빌더조쉬가 실험한 사례를 보면 이렇게 진행돼요.

**테스트 1: AI 트렌드 PPT**
- 주제: "2025 AI 트렌드"
- 리서치 에이전트가 최신 자료 검색
- 정리 에이전트가 5개 슬라이드로 구조화
- 디자인 스킬이 브랜드 컬러 적용
- pptx 스킬이 최종 파일 생성
- 소요 시간: 약 3분

**테스트 2: 자기소개 PPT**
- 주제: "빌더조쉬 소개"
- 기존 웹사이트/SNS에서 정보 수집
- 6개 슬라이드로 구성
- 디자인 개선 피드백 반영
- 소요 시간: 약 4분

**테스트 3: 다른 크리에이터 PPT**
- 주제: "옵티다운 소개"
- 공개된 정보로 자동 작성
- 일관된 디자인 유지
- 소요 시간: 약 3분

사람이 하면 각각 2-3시간 걸릴 작업을 3-4분 만에 끝냈어요. 물론 최종 검토와 수정은 필요하지만, 80-90%는 자동화된 거죠.

## 팀에서 활용할 때 더 강력해지는 이유

개인이 쓰기에도 좋지만, 이 시스템은 팀에서 쓸 때 진짜 가치가 나와요.

**브랜드 일관성 유지**
- 모든 팀원이 같은 디자인 가이드라인 사용
- 폰트, 컬러, 레이아웃이 자동으로 통일
- 클라이언트에게 일관된 품질 전달

**산출물 표준화**
- 신입이든 경력이든 같은 퀄리티
- 온보딩 시간 단축
- 품질 관리 부담 감소

**노하우 축적**
- 잘 만든 프롬프트를 팀 자산으로 공유
- 개선사항을 지속적으로 업데이트
- 시간이 갈수록 품질 향상

**주의할 점:**
- 초기 세팅에 시간 투자 필요 (프롬프트 작성, 테스트)
- 브랜드 가이드라인이 명확해야 효과적
- 팀원 교육과 합의 과정 중요

스타트업이나 에이전시처럼 산출물이 많은 조직에서 특히 유용해요. 한 번 세팅해 두면 계속 재사용할 수 있으니까요.

## 자주 묻는 질문 (FAQ)

### Q1. Claude Code 사용에 비용이 얼마나 드나요?
A. Claude Code는 토큰 기반 과금이에요. SubAgent를 많이 쓰면 비용이 올라가지만, Skills를 적절히 활용하면 비용을 크게 줄일 수 있어요. PPT 한 개당 약 2,000-5,000원 정도 예상하면 됩니다. 사람이 직접 만드는 시간 비용을 생각하면 충분히 저렴해요.

### Q2. 코딩을 몰라도 할 수 있나요?
A. 네, 가능해요. Claude Code는 자연어로 지시하면 되니까 코딩 지식이 필수는 아니에요. 다만 Cursor 같은 에디터와 터미널 사용법은 익혀야 해요. 빌더조쉬 영상을 따라 하면 비개발자도 충분히 세팅할 수 있답니다.

### Q3. 기존 회사 PPT 템플릿을 적용할 수 있나요?
A. 물론이죠! 브랜드 가이드라인(폰트, 컬러, 로고 등)을 디자인 스킬 프롬프트에 명시하면 자동으로 적용돼요. 회사 전용 템플릿을 만들어 두면 팀 전체가 일관된 디자인으로 작업할 수 있어요.

### Q4. SubAgent와 Skills 중 뭘 써야 할지 어떻게 판단하나요?
A. 간단해요. "이 작업에 상황 판단이 필요한가?"를 물어보세요. 판단이 필요하면 SubAgent, 정해진 규칙대로만 하면 되면 Skills예요. 예를 들어 "신뢰도 높은 자료 찾기"는 SubAgent, "찾은 자료를 JSON으로 변환"은 Skills죠.

### Q5. n8n이나 Make 같은 도구와 뭐가 다른가요?
A. n8n/Make는 미리 정의된 노드를 연결하는 방식이라 정해진 작업만 가능해요. Claude Code는 자연어로 지시하면 AI가 방법을 찾아내니까 훨씬 유연해요. 특히 에이전트끼리 소통하며 복잡한 업무를 처리하는 건 Claude Code만의 강점이에요.
