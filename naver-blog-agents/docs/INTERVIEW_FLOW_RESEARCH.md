# 글또 품질 향상을 위한 인터뷰 플로우 연구

## 연구 배경

현재 글또 시스템은 기본적인 정보(주제, 유형, 키워드 등)만으로 콘텐츠를 생성합니다.
그러나 각 작가 에이전트의 프롬프트를 분석한 결과, **콘텐츠 유형별로 훨씬 더 많은 정보가 필요**함을 확인했습니다.

### 현재 수집 정보 (ContentRequest)

```typescript
interface ContentRequest {
  topic: string;              // 주제
  type: ContentType;          // 유형 (info, food, review 등)
  keywords?: string[];        // 키워드
  targetAudience?: string;    // 타겟 독자
  tone?: ContentTone;         // 톤
  additionalContext?: string; // 추가 컨텍스트
  referenceUrls?: string[];   // 참고 URL
  referenceData?: object;     // 참조 데이터
}
```

### 문제점

1. **구조화되지 않은 정보**: `additionalContext`에 모든 것을 담으려 함
2. **콘텐츠 유형별 특화 정보 부재**: 맛집에는 위치/가격, 리뷰에는 스펙/사용기간 등
3. **실제 경험 정보 부족**: AI가 "실제 방문한 것처럼" 쓰려면 구체적 경험 필요
4. **글 스타일 선호도 미반영**: 글 길이, 이모지 사용, 사진 위치 등

---

## 제안: 인터뷰 플로우 시스템

사용자에게 **콘텐츠 유형별 맞춤 질문**을 하여 품질 높은 정보를 수집합니다.

### 인터뷰 단계

```
[1단계] 기본 정보 확인
    ↓
[2단계] 콘텐츠 유형별 심화 질문
    ↓
[3단계] 경험/사실 정보 수집
    ↓
[4단계] 스타일 선호도 확인
    ↓
[콘텐츠 생성]
```

---

## 1단계: 기본 정보 확인

모든 콘텐츠 유형에 공통으로 적용됩니다.

| 질문 | 목적 | 예시 답변 |
|-----|-----|---------|
| 어떤 주제로 블로그 글을 쓸까요? | 핵심 주제 파악 | "강남역 파스타 맛집" |
| 어떤 유형의 글인가요? | 담당 작가 배정 | 맛집리뷰, 제품리뷰 등 |
| 타겟 독자는 누구인가요? | 톤/내용 조절 | "20-30대 직장인" |
| 글의 목적은 무엇인가요? | 핵심 메시지 도출 | 정보 제공 / 추천 / 후기 공유 |

## 2단계: 콘텐츠 유형별 심화 질문

### 맛집리뷰 (food) - 하린

```yaml
위치_정보:
  - 정확한 주소 또는 위치 (ex: 강남역 3번 출구 도보 5분)
  - 찾아가기 어려운지, 주차 가능한지

가게_정보:
  - 가게 이름
  - 업종 (한식/양식/중식/일식/카페 등)
  - 분위기 (데이트 코스/가족 모임/혼밥 등)
  - 영업시간 (알고 있다면)

음식_정보:
  - 주문한 메뉴와 가격
  - 대표 메뉴는 무엇인지
  - 맛에 대한 간단한 평가 (맵기, 달기, 식감 등)

경험_정보:
  - 방문 날짜/시간
  - 동행인 (혼자/친구/연인/가족)
  - 웨이팅 있었는지
  - 서비스 만족도
  - 재방문 의사
```

**예시 질문:**
1. "어떤 메뉴를 드셨고, 가격은 얼마였나요?"
2. "분위기는 어땠나요? (데이트/가족/혼밥 등)"
3. "웨이팅이 있었나요? 주차는 가능했나요?"
4. "재방문 의사가 있으신가요? 별점을 준다면?"

---

### 제품리뷰 (review/tech) - 태현/민석

```yaml
제품_정보:
  - 정확한 제품명과 모델명
  - 구매 가격 및 구매처
  - 구매 시기

사용_정보:
  - 사용 기간 (일주일/한달/6개월 등)
  - 주 사용 용도
  - 사용 빈도

경험_평가:
  - 장점 3가지
  - 단점 2가지
  - 기대 vs 실제 차이
  - 추천 대상
  - 비추천 대상

비교_정보:
  - 이전에 사용한 유사 제품
  - 비교 대상 제품
  - 최종 선택 이유
```

**예시 질문:**
1. "제품명과 구매 가격을 알려주세요."
2. "얼마나 사용해보셨나요? 주로 어떤 용도로 사용하시나요?"
3. "가장 마음에 드는 점과 아쉬운 점을 각각 알려주세요."
4. "비슷한 다른 제품과 비교한다면?"

### 여행 (travel) - 유진

```yaml
여행_기본정보:
  - 여행지 (국가/도시)
  - 여행 기간 (N박 M일)
  - 여행 시기 (계절/날씨)
  - 동행 (혼자/커플/가족/친구)

일정_정보:
  - 방문한 주요 명소 (2-5개)
  - 추천 코스 (있다면)
  - 각 장소에서 한 활동

숙소_정보:
  - 숙소 이름/유형
  - 위치 장점
  - 가격대
  - 만족도

비용_정보:
  - 총 예산
  - 항목별 대략적 비용 (숙박/식사/입장료 등)

경험_정보:
  - 가장 인상적인 경험
  - 아쉬웠던 점
  - 여행 팁
```

**예시 질문:**
1. "몇 박 몇 일 여행이었고, 누구와 함께 가셨나요?"
2. "어떤 명소를 방문하셨나요? 가장 좋았던 곳은?"
3. "숙소는 어땠나요? 추천할 만한가요?"
4. "총 경비는 대략 얼마 정도 들었나요?"

### 정보성 (info) - 현우 선생님

```yaml
주제_범위:
  - 다루고 싶은 세부 주제
  - 다루지 않을 내용 (범위 한정)
  - 깊이 (입문/중급/고급)

독자_수준:
  - 사전 지식 수준
  - 용어 설명 필요 여부
  - 실습/예제 포함 여부

참고_자료:
  - 인용하고 싶은 출처
  - 참고할 만한 URL
  - 공식 자료 (통계, 논문 등)

구성_선호:
  - 목차 구성 (단계별/주제별)
  - 예시 포함 여부
  - 요약/정리 형식
```

**예시 질문:**
1. "독자가 이미 알고 있다고 가정할 내용은 무엇인가요?"
2. "반드시 포함해야 할 핵심 내용은 무엇인가요?"
3. "참고할 만한 자료나 출처가 있나요?"
4. "글의 깊이는 어느 정도가 좋을까요? (입문/중급/고급)"

### 마케팅 (marketing) - 지은 언니

```yaml
브랜드_정보:
  - 브랜드/제품/서비스명
  - 핵심 특징 (USP)
  - 타겟 고객

협찬_여부:
  - 협찬/광고 여부
  - 협찬 조건 (금액, 제품 제공 등)
  - 필수 포함 문구

메시지:
  - 전달하고 싶은 핵심 메시지
  - 강조하고 싶은 혜택
  - CTA (Call-to-Action)

톤_스타일:
  - 분위기 (럭셔리/친근/전문적)
  - 감성적 vs 정보적 비율
  - 스토리텔링 요소
```

**예시 질문:**
1. "어떤 브랜드/제품/서비스인가요? 핵심 특징은?"
2. "협찬 콘텐츠인가요? 필수 포함 문구가 있나요?"
3. "독자에게 전달하고 싶은 핵심 메시지는?"
4. "글의 톤은 어떻게 할까요? (친근/전문적/럭셔리)"

### 육아 (parenting) - 예원맘

```yaml
아이_정보:
  - 아이 나이/개월 수
  - 성별 (해당 시)
  - 발달 단계 (신생아/영아/유아 등)

주제_구체화:
  - 육아템 추천 / 육아 팁 / 성장 기록 / 고민 해결
  - 특정 상황 (이유식 시작, 어린이집 적응 등)
  - 경험담 vs 정보 제공

경험_정보:
  - 직접 사용/경험 여부
  - 사용 기간
  - 효과/결과

주의사항:
  - 안전 관련 정보
  - 의료 면책 필요 여부
```

**예시 질문:**
1. "아이가 몇 개월/몇 살인가요?"
2. "어떤 상황에서의 경험을 공유하고 싶으신가요?"

4. "다른 부모님들에게 가장 도움이 될 정보는 무엇일까요?"

```yaml
주제_분류:
```

## 3단계: 경험/사실 정보 수집

**실제 경험이 있는 경우:**

```yaml
직접_경험:
  - 방문/사용 날짜
  - 구체적인 에피소드
  - 사진 촬영 여부
  - 기억에 남는 디테일

감정_경험:
  - 첫인상
  - 가장 좋았던 순간
  - 아쉬웠던 점
  - 전체 만족도 (1-10)
```

**간접 정보인 경우:**

```yaml
정보_출처:
  - 참고한 자료/URL
  - 신뢰도 평가
  - 팩트체크 필요 항목
  - 개인 의견 vs 객관적 정보 구분
```

## 4단계: 스타일 선호도 확인

```yaml
글_스타일:
  - 글 길이 (짧게/보통/길게)
  - 문체 (formal/casual/friendly)
  - 이모지 사용 (많이/적당히/없이)
  - 목록 vs 서술형 선호

구성_선호:
  - 제목 스타일 (질문형/숫자형/선언형)
  - 소제목 개수 (3-5개/5-7개/7개 이상)
  - 사진 위치 안내 포함 여부
  - CTA 강도 (약하게/보통/강하게)

SEO_관련:
  - 목표 키워드 (있다면)
  - 검색 노출 중요도
  - 태그 개수 선호
```

### 방안 1: Claude Code 에이전트 대화형

```

글또: "맛집 리뷰를 작성해드릴게요! 몇 가지만 여쭤볼게요.

📍 위치/가게 정보
1. 가게 이름과 정확한 위치가 어떻게 되나요?
2. 어떤 종류의 음식점인가요? (한식/양식/일식 등)

🍽️ 음식 정보
3. 어떤 메뉴를 드셨고, 가격은 얼마였나요?
4. 맛은 어떠셨나요? 간단히 평가해주세요.

💡 경험 정보
5. 누구와 함께 가셨나요? 분위기는 어땠나요?
6. 재방문 의사가 있으신가요?

원하시는 정보만 답변해주셔도 됩니다!"
```


## 구현 우선순위

## 기술 구현 방안

### 1. 새로운 타입 정의

```typescript
// types/interview.ts

export interface InterviewData {
  type: ContentType;
  questions: InterviewQuestion[];
  answers: Record<string, unknown>;
}

export interface InterviewQuestion {
  id: string;
  category: string;
  question: string;
  required: boolean;
  inputType: 'text' | 'number' | 'select' | 'multiselect' | 'rating';
  options?: string[];
}

// 콘텐츠 유형별 인터뷰 스키마
export const INTERVIEW_SCHEMAS: Record<ContentType, InterviewQuestion[]> = {
  food: [
    { id: 'restaurant_name', category: '가게정보', question: '가게 이름', required: true, inputType: 'text' },
    { id: 'location', category: '가게정보', question: '위치 (역/출구 기준)', required: true, inputType: 'text' },
    // ...
  ],
  review: [
    { id: 'product_name', category: '제품정보', question: '제품명', required: true, inputType: 'text' },
    // ...
  ],
  // ...
};
```

### 3. 인터뷰어 에이전트 추가

```typescript
// agents/interviewer.ts

export class InterviewerAgent {
  async conductInterview(type: ContentType): Promise<InterviewData> {
    const schema = INTERVIEW_SCHEMAS[type];
    const answers: Record<string, unknown> = {};

    for (const question of schema) {
      const answer = await this.askQuestion(question);
      if (answer || question.required) {
        answers[question.id] = answer;
      }
    }

    return { type, questions: schema, answers };
  }
}
```
