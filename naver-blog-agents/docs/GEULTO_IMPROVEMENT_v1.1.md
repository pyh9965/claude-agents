## 3. 개선 과제 상세

### 3.1 [HIGH] SEO 전처리 시스템 도입

**현재 문제**
```
Planning(민준) → Research(수빈) → Writing(작가) → SEO(준서)
                                                    ↑
                                              SEO가 후처리만 담당
```

**개선안**
```
Planning(민준) + SEO 키워드 선정(준서) → Research → Writing → SEO 최종 검증
       ↑
  기획 단계부터 핵심 키워드 3~5개 확정
```

**구현 방법**
- `src/workflow/orchestrator.ts` 수정
- Planning 단계에서 SEO Expert를 먼저 호출하여 키워드 리스트 생성
- 해당 키워드를 Plan에 포함하여 작가에게 전달

**예상 효과**
- 제목에 핵심 키워드 자동 포함
- 소제목에 키워드 + 의미 조합

### 3.2 [HIGH] 표(Table) 형식 강화

**현재 출력**
```markdown
- 59A 타입: 11억 6,900만 원
- 84D 타입: 16억 4,100만 원
```

**개선 출력**
```markdown
| 타입 | 전용면적 | 세대수 | 최고 분양가 | 경쟁률 |
|-----|---------|-------|-----------|--------|
| 59A | 59.83㎡ | 176 | 11.69억 | 5.36:1 |
| 84D | 84.99㎡ | 32 | 16.41억 | 32.00:1 |
```

**구현 방법**
- `prompts/writers/*.md` 수정
- "핵심 수치 데이터는 반드시 표(table) 형식으로 정리" 지침 추가
- 필수 표 목록 명시:
  1. 사업 개요 표
  2. 타입별 세대수/경쟁률 표
  3. 타입별 분양가 표

### 3.3 [MEDIUM] 자금 계산기 추가

**현재**: 자금 계획 언급만 있음
**개선**: 구체적인 계산 예시 제공

**구현 방법**
- `src/agents/researcher.ts`에 자금 계산 로직 추가

```typescript
interface FundingCalculation {
  totalPrice: number;           // 분양가
  ltvRatio: number;             // LTV 비율 (기본 60%)
  loanAmount: number;           // 대출 가능액
  selfFunding: number;          // 자기자본 필요액
  monthlyInterest: number;      // 월 이자 (금리 5% 가정)
}

function calculateFunding(price: number): FundingCalculation {
  const ltvRatio = 0.6;
  const annualRate = 0.05;
  const loanAmount = price * ltvRatio;
  return {
    totalPrice: price,
    ltvRatio,
    loanAmount,
    selfFunding: price - loanAmount,
    monthlyInterest: Math.round(loanAmount * annualRate / 12),
  };
}
```

**예상 출력**
```markdown
### 💰 84D 타입 자금 계획 예시 (16.4억 기준)
| 항목 | 금액 |
|-----|------|
| 분양가 | 16.4억 |
| 대출 가능액 (LTV 60%) | 9.84억 |
| 자기자본 필요 | 6.56억 |
| 중도금 이자 (연 5% 가정) | 월 약 410만 원 |
```
