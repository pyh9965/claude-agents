## 1. 소개

### 1.1 글또란?

글또는 네이버 블로그 콘텐츠를 자동으로 생성하는 **12인 AI 에이전트 팀**입니다. 각 에이전트는 고유한 페르소나와 전문성을 가지고 있어, 마치 실제 콘텐츠 제작팀처럼 협업하여 고품질 블로그 글을 작성합니다.

### 1.2 주요 특징

| 특징 | 설명 |
|------|------|
| **12명의 전문 에이전트** | 기획자, 리서처, 작가 8명, 편집자, SEO 전문가 |
| **8가지 콘텐츠 유형** | 정보성, 마케팅, 리뷰, 맛집, 여행, 테크, 라이프스타일, 육아 |
| **병렬 처리** | 편집과 SEO 작업을 동시에 처리하여 시간 단축 |
| **3가지 출력 포맷** | 네이버 HTML, 마크다운, JSON |
| **Self-Reflection** | 작가가 스스로 글을 검토하고 개선 |
| **팩트체크 지원** | 참조 데이터 기반 정확한 정보 작성 |

### 1.3 기술 스택

- **런타임**: Node.js 18+
- **언어**: TypeScript
- **AI 엔진**: Google Gemini API (gemini-1.5-flash)
- **패키지 매니저**: npm

## 2. 설치 및 환경 설정

### 2.1 사전 요구사항

- Node.js 18.0.0 이상
- npm 또는 yarn
- Google Gemini API 키 ([발급 방법](https://ai.google.dev/))

### 2.2 설치

```bash
# 프로젝트 디렉토리로 이동
cd naver-blog-agents

# 의존성 설치
npm install

# TypeScript 빌드
npm run build
```

### 2.3 환경 변수 설정

`.env` 파일을 생성하고 다음 내용을 입력합니다:

```env
# ==========================================
# 필수 설정
# ==========================================

# Google Gemini API 키 (필수)
GEMINI_API_KEY=AIzaSy...your_key_here

# ==========================================
# 선택 설정
# ==========================================

# 네이버 API (고급 검색 기능용)
NAVER_CLIENT_ID=your_naver_client_id
NAVER_CLIENT_SECRET=your_naver_client_secret

# 출력 디렉토리 (기본값: ./output)
OUTPUT_DIR=./output

# 로그 레벨 (debug, info, warn, error)
LOG_LEVEL=info
```

### 2.4 API 키 발급 방법

#### Google Gemini API 키

1. [Google AI Studio](https://ai.google.dev/) 접속
2. "Get API key" 클릭
3. 새 프로젝트 생성 또는 기존 프로젝트 선택
4. API 키 복사 → `.env` 파일에 붙여넣기

## 4. CLI 명령어 완벽 가이드

### 4.1 generate - 콘텐츠 생성

가장 핵심적인 명령어입니다.

```bash
npm run cli -- generate [옵션]
```

#### 필수 옵션

| 옵션 | 단축 | 설명 | 예시 |
|------|------|------|------|
| `--topic` | `-t` | 콘텐츠 주제 | "강남 맛집 추천" |
| `--type` | `-T` | 콘텐츠 유형 | info, marketing, review, food, travel, tech, lifestyle, parenting |

#### 선택 옵션

| 옵션 | 단축 | 설명 | 기본값 |
|------|------|------|--------|
| `--keywords` | `-k` | 키워드 (쉼표 구분) | - |
| `--audience` | `-a` | 타겟 독자층 | - |
| `--tone` | - | 톤 (formal/casual/friendly) | friendly |
| `--context` | `-c` | 추가 컨텍스트 | - |
| `--data` | `-d` | 참조 데이터 JSON 파일 | - |
| `--apt` | - | 청약홈 DB에서 아파트 검색 | - |
| `--crawl` | - | 분양 홈페이지 크롤링 | false |
| `--output` | `-o` | 출력 디렉토리 | ./output/주제-날짜 |
| `--timeout` | - | 타임아웃 (밀리초) | 600000 |
| `--verbose` | - | 상세 로그 출력 | false |

#### 사용 예시

```bash
# 기본 정보성 콘텐츠
npm run cli -- generate -t "노트북 구매 가이드" -T info

# 키워드 포함 맛집 리뷰
npm run cli -- generate -t "강남역 파스타 맛집" -T food -k "강남역,점심,데이트,파스타"

# 타겟 독자 지정 마케팅 콘텐츠
npm run cli -- generate -t "새로운 스킨케어 브랜드" -T marketing -a "20-30대 피부 고민 있는 여성"

# 추가 컨텍스트 포함 여행 콘텐츠
npm run cli -- generate -t "제주도 3박4일 코스" -T travel -c "신혼여행, 럭셔리 호텔 중심으로"

# 참조 데이터 파일 사용
npm run cli -- generate -t "래미안 신축 아파트 분석" -T info -d "./data/raemian.json"

# 청약홈 DB 연동 (분양 아파트 분석용)
npm run cli -- generate -t "래미안 원페를 분석" -T info --apt "래미안원페를" --crawl

# 상세 로그 출력
npm run cli -- generate -t "맥북 프로 M4 리뷰" -T tech --verbose
```

## 7. 에이전트 팀 소개

### 7.1 팀 구성도

```
                        ┌─────────────┐
                        │  민준 팀장  │
                        │   (기획자)  │
                        └──────┬──────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
          ▼                    ▼                    ▼
    ┌──────────┐        ┌───────────┐        ┌──────────┐
    │   수빈   │        │  작가팀   │        │  QA팀    │
    │ (리서처) │        │  (8명)    │        │  (2명)   │
    └──────────┘        └─────┬─────┘        └────┬─────┘
                              │                   │
        ┌─────────────────────┼─────────────────┐ │
        ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼ ▼
      현우  지은  태현  하린  유진  민석  수아 예원맘 서연 준서
      (정보)(마케팅)(리뷰)(맛집)(여행)(테크)(라이프)(육아)(편집)(SEO)
```

### 7.2 에이전트 상세 정보

#### 기획 단계

| 에이전트 | 나이 | 역할 | 특징 | 말투 |
|---------|------|------|------|------|
| 📋 민준 팀장 | 38세 | 기획자 | 차분하고 전략적인 베테랑 | "전체적인 방향은~" |
| 🔍 수빈 | 29세 | 리서처 | 꼼꼼한 팩트체커 | "확인해본 결과~" |

#### 작가팀

| 에이전트 | 나이 | 담당 | 특징 | 문체 |
|---------|------|------|------|------|

## 8. 워크플로우 이해하기

### 8.1 전체 흐름

```
[입력] → [기획] → [리서치] → [작성] → [편집 + SEO] → [포맷팅] → [출력]
         민준     수빈       작가     서연    준서      자동
```

### 8.2 단계별 상세 설명

| 단계 | 담당 | 설명 | 예상 시간 |
|------|------|------|----------|
| 1. Planning | 민준 팀장 | 콘텐츠 전략 수립, 아웃라인 설계, 작가 배정 | ~25초 |
| 2. Research | 수빈 | 정보 수집, 팩트체크, 참조 데이터 활용 | ~25초 |
| 3. Writing | 담당 작가 | 초안 작성 + Self-Reflection으로 자체 검토 | ~40초 |
| 4. Editing | 서연 실장 | 맞춤법, 톤 일관성, 가독성 검토 | ~20초 |
| 5. SEO | 준서 | 키워드, 제목, 메타 설명 최적화 | ~20초 |
| 6. Formatting | 시스템 | HTML, MD, JSON 변환 | 즉시 |

> **Note**: Editing과 SEO는 병렬로 실행되어 시간을 절약합니다.

### 8.3 Self-Reflection 기능

작가 에이전트는 초안 작성 후 스스로 다음을 검토합니다:
- 글의 흐름과 논리성
- 누락된 정보 확인
- 독자 관점에서의 이해도
- 개선점 파악 및 수정

## 9. 출력 파일 활용법

### 9.1 출력 파일 구조

```
output/강남-맛집-추천-2025-01-31/
├── content.html      # 네이버 스마트에디터용 HTML
├── content.md        # 범용 마크다운
├── content.json      # 구조화된 JSON
└── metadata.json     # 메타데이터
```

### 9.2 content.html - 네이버 블로그 업로드

**사용 방법:**

1. `content.html` 파일을 텍스트 에디터로 열기
2. 전체 내용 복사 (Ctrl+A → Ctrl+C)
3. 네이버 블로그 글쓰기 페이지 접속
4. "HTML" 모드로 전환 (에디터 상단)
5. 붙여넣기 (Ctrl+V)
6. "에디터" 모드로 돌아가서 미리보기 확인
7. 이미지 추가 및 최종 수정 후 발행

### 9.3 content.md - 마크다운

**활용 방법:**
- Notion, Obsidian에서 바로 사용
- GitHub, GitLab README로 활용
- 다른 블로그 플랫폼 (Velog, Tistory)에 사용
- 추가 편집 후 재변환

### 9.4 content.json - 프로그래밍 연동

## 10. 고급 기능

### 10.1 참조 데이터 활용

정확한 정보가 필요한 콘텐츠는 참조 데이터를 제공하면 팩트체크된 글을 작성합니다.

```bash
# JSON 파일로 참조 데이터 제공
npm run cli -- generate -t "래미안 신축 아파트 분석" -T info -d "./data/apt-info.json"
```

**참조 데이터 예시 (apt-info.json):**
```json
{
  "아파트명": "래미안 원펠리스",
  "위치": "서울시 강남구",
  "세대수": 1500,
  "입주예정": "2026년 3월",
  "분양가": "평당 5,000만원",
  "특징": ["지하철 역세권", "대형 커뮤니티", "학군 우수"]
}
```

### 10.2 청약홈 DB 연동

분양 아파트 콘텐츠 작성 시 청약홈 데이터를 자동으로 활용합니다.

```bash
# 청약홈 DB에서 아파트 검색
npm run cli -- generate -t "래미안 원펠리스 분석" -T info --apt "래미안원펠리스"

# 분양 홈페이지 크롤링까지 추가
npm run cli -- generate -t "래미안 원펠리스 분석" -T info --apt "래미안원펠리스" --crawl
```

### 10.3 타임아웃 조절

복잡한 주제나 긴 콘텐츠는 타임아웃을 늘려줍니다.

```bash
# 15분 타임아웃
npm run cli -- generate -t "2025 종합 가이드" -T info --timeout 900000
```

### 10.4 상세 로그 출력

문제 해결이나 진행 상황 확인 시 사용합니다.

```typescript
import { createBlogContent } from 'geulto';

async function main() {
```

## 12. 문제 해결

### 12.1 일반적인 오류

#### API 키 오류
```
Error: GEMINI_API_KEY 환경 변수가 설정되지 않았습니다.
```
**해결:** `.env` 파일에 올바른 API 키 설정

#### 타임아웃 오류
```
Error: 작성 단계 시간 초과
```
**해결:** `--timeout` 옵션으로 시간 늘리기
```bash
npm run cli -- generate -t "주제" -T info --timeout 900000
```

#### 빌드 오류
```
Error: Cannot find module 'dist/cli.js'
```
**해결:** 빌드 실행
```bash
npm run build
```

### 12.2 콘텐츠 품질 문제

#### 내용이 너무 짧음
- 키워드 추가: `-k "키워드1,키워드2,키워드3"`
- 컨텍스트 추가: `-c "더 상세하게, 각 항목별로 자세히"`

#### 톤이 맞지 않음
- `--tone` 옵션 변경: `formal`, `casual`, `friendly`

#### 정보가 부정확함
- 참조 데이터 제공: `-d "./data.json"`

### 12.3 디버깅

```bash
# 상세 로그 출력
npm run cli -- generate -t "테스트" -T info --verbose

# 로그 레벨 변경 (.env)
LOG_LEVEL=debug
```

## 13. FAQ

### Q1. 콘텐츠 생성에 얼마나 걸리나요?
평균 2-3분 정도 소요됩니다. 복잡한 주제는 더 걸릴 수 있습니다.

### Q2. 한 번에 여러 콘텐츠를 생성할 수 있나요?
현재는 한 번에 하나의 콘텐츠만 생성할 수 있습니다. 여러 콘텐츠가 필요하면 순차적으로 실행하세요.

### Q3. 생성된 콘텐츠를 수정해도 되나요?
물론입니다! 글또가 생성한 콘텐츠는 초안입니다. 이미지 추가, 개인 경험 추가, 문구 수정 등 자유롭게 편집하세요.

### Q4. 이미지는 자동으로 추가되나요?
아니요. 글또는 텍스트 콘텐츠만 생성합니다. 이미지는 직접 추가해야 합니다.

### Q5. 어떤 콘텐츠 유형을 선택해야 할지 모르겠어요.

| 주제 예시 | 추천 유형 |
|----------|----------|
| ~하는 방법, 가이드, 비교 | info |
| 브랜드, 협찬, 이벤트 | marketing |
| IT, 가전 리뷰 | review 또는 tech |
| 맛집, 카페 | food |
| 국내/해외 여행 | travel |
| 일상, 인테리어, 루틴 | lifestyle |
| 육아, 아이 관련 | parenting |

### Q6. API 비용이 발생하나요?
Gemini API는 무료 티어가 있습니다. 일일 한도 내에서는 무료로 사용 가능합니다. 자세한 내용은 Google AI Studio에서 확인하세요.

### Q7. 네이버 API 키가 꼭 필요한가요?
아니요. 네이버 API는 선택 사항입니다. 없어도 기본 기능은 모두 사용할 수 있습니다.
